# Firebase Setup Guide for Kavach

**Updated**: April 5, 2026  
**Architecture**: Serverless Firebase Backend  
**Cloud Deployment**: Google Cloud Platform (GCP)  
**Mobile Distribution**: Firebase App Distribution + Google Play Store

---

## 🚀 Quick Setup (5 Minutes)

### Step 1: Create Firebase Project

```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Create new project
firebase projects create kavach-production --display-name "Kavach"

# OR use existing GCP project
firebase init --project=kavach-production
```

### Step 2: Connect to Flutter App

In `mobile/lib/main.dart`:
```dart
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart'; // Generated by FlutterFire CLI

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );
  runApp(const KavachApp());
}
```

### Step 3: Run FlutterFire CLI

```bash
cd mobile
dart pub global activate flutterfire_cli
flutterfire configure --project=kavach-production
# Automatically generates firebase_options.dart
```

## 🔥 Firebase Services Setup

### **1. Authentication (Firebase Auth)**

```bash
# Enable in Firebase Console:
# → Authentication → Sign-in method
# → Enable: Phone, Google, Email/Password
```

**In Flutter**:
```dart
import 'package:firebase_auth/firebase_auth.dart';

class AuthService {
  final FirebaseAuth _auth = FirebaseAuth.instance;

  // Phone OTP authentication
  Future<void> signInWithPhone(String phoneNumber) async {
    await _auth.verifyPhoneNumber(
      phoneNumber: phoneNumber,
      verificationCompleted: (PhoneAuthCredential credential) async {
        await _auth.signInWithCredential(credential);
      },
      codeSent: (String verificationId, int? resendToken) {
        // Show OTP input screen
      },
      codeAutoRetrievalTimeout: (String verificationId) {},
      verificationFailed: (FirebaseAuthException e) {
        print('Auth error: ${e.message}');
      },
    );
  }
}
```

### **2. Firestore Database**

```bash
# Enable in Firebase Console:
# → Firestore Database → Create database
# → Select region: asia-south1 (India)
# → Start in production mode
# → Deploy security rules
```

**Deploy Rules**:
```bash
cd backend/firestore_rules
firebase deploy --only firestore:rules --project=kavach-production
```

**Deploy Indexes**:
```bash
firebase deploy --only firestore:indexes --project=kavach-production
```

**In Flutter**:
```dart
import 'package:cloud_firestore/cloud_firestore.dart';

final FirebaseFirestore db = FirebaseFirestore.instance;

// Create incident
await db.collection('incidents').add({
  'userId': user.uid,
  'type': 'crash',
  'location': {
    'latitude': location.latitude,
    'longitude': location.longitude,
  },
  'createdAt': FieldValue.serverTimestamp(),
});

// Listen to real-time updates
db.collection('incidents')
  .where('userId', isEqualTo: user.uid)
  .snapshots()
  .listen((snapshot) {
    for (var doc in snapshot.docs) {
      print('Incident: ${doc.data()}');
    }
  });
```

## 🧠 Gemini AI Integration

### Setup Gemini API

```bash
# 1. Enable Vertex AI API in GCP Console
gcloud services enable aiplatform.googleapis.com

# 2. Create API key
gcloud alpha services api-keys create \
  --display-name="Kavach Gemini API Key" \
  --api-target=generativeai.googleapis.com
```

### In Flutter

```dart
import 'package:google_generative_ai/google_generative_ai.dart';

// Initialize Gemini
final gemini = GenerativeModel(
  model: 'gemini-pro',
  apiKey: Platform.environment['GEMINI_API_KEY'] ?? 'YOUR_API_KEY',
);

// Analyze crash with Gemini
final response = await gemini.generateContent([
  Content.text('''
Analyze this crash: Impact ${crashForce}G at ${speed}km/h.
Impact direction: $direction.
Provide medical guidance and hospital recommendation.
Format as JSON.
'''),
]);

print(response.text);
```

```bash

flutter build apk --release

# Upload to Play Console
# → Your app → Release → Production
# → Upload APK
# → Review and publish
```

```bash
# Upload APK to Cloud Storage
gsutil cp build/app/outputs/apk/release/app-release.apk \
  gs://kavach-storage.appspot.com/releases/app-v1.0.0.apk

gsutil signurl key.json gs://kavach-storage.appspot.com/releases/app-v1.0.0.apk
```

## 🚀 CI/CD Pipeline with Cloud Build

### **Setup Cloud Build**

```bash
# Connect GitHub repository
gcloud builds connect --repository-name=Kavach --repository-owner=preetbiswas12

# Create build trigger
gcloud builds triggers create github \
  --repo-name=Kavach \
  --repo-owner=preetbiswas12 \
  --branch-pattern="^main$" \
  --build-config=backend/gcp_deployment/cloudbuild.yaml
```

### **Automated Pipeline**

```yaml
# cloudbuild.yaml
steps:
  # Build Flutter APK
  - name: 'gcr.io/cloud-builders/docker'
    args: ['build', '-t', 'gcr.io/$PROJECT_ID/kavach:$COMMIT_SHA', '.']
  
  # Deploy Cloud Functions
  - name: 'gcr.io/firebase-tools/firebase-tools'
    args: ['deploy', '--only', 'functions']
  
  # Deploy Firestore rules
  - name: 'gcr.io/firebase-tools/firebase-tools'
    args: ['deploy', '--only', 'firestore:rules']
  
  # Distribute APK
  - name: 'gcr.io/firebase-tools/firebase-tools'
    args: ['app-distribution:distribute', 'app.apk']
```

## 🔐 Security Best Practices

```bash
# 1. Store API keys in Cloud Secret Manager
gcloud secrets create gemini-api-key --data-file=key.txt

# 2. Use least privilege IAM roles
gcloud projects add-iam-policy-binding kavach-production \
  --member=serviceAccount:firebase-adminsdk@kavach-production.iam.gserviceaccount.com \
  --role=roles/firestore.serviceAgent

# 3. Enable audit logging
gcloud logging create /projects/kavach-production/logs/firestore-audit --bucket=kavach-audit-logs
```
